# DevScale Anomaly Playbook

Use this playbook when reviewing week-over-week movement.

- Treat a 15% or larger week-over-week move as notable.
- Treat a 25% or larger move as a headline change.
- Flow metrics use weekly sums: projectsDelivered, revenue, tasksCompleted.
- Stock metrics use end-of-period values: clientsActive, clientSatisfaction, teamUtilization.
- Delivery rate is projectsDelivered / clientsActive.
- Task efficiency is tasksCompleted / projectsDelivered.
- Client satisfaction is a score (0-100), not a count.
- If multiple operational metrics move together, describe the shared direction
  instead of inventing a causal story.
